Introduction to HTML

What is HTML (HyperText Markup Language)?

HTML, or HyperText Markup Language, is the standard language used to create and structure content on the web. It consists of a series of elements represented by tags that tell the browser how to display content such as text, images, links, and other multimedia. Every webpage you visit is built using HTML at its core

Importance of HTML for Structuring Web Pages

HTML is the foundation of all web pages, providing the essential structure that defines the hierarchy of elements like headings, paragraphs, links, and images. Without HTML, browsers would not know how to render the content of a page. It also enables the integration of CSS (Cascading Style Sheets) for styling and JavaScript for interactivity, but the layout and basic structure come from HTML.

Basic Document Structure

Every HTML document follows a standard structure that helps browsers interpret and display the content. Below is a breakdown of the basic HTML document structure:

  • <!DOCTYPE html>: Declares the document type and version of HTML.
  • <html>: Declares the document type and version of HTML.
  • <head>: Declares the document type and version of HTML.
  • <body>: Declares the document type and version of HTML. images, and more

Example

                    
                        <!DOCTYPE html>
                        <html>
                            <head>
                                <title>My First Page</title>
                            <head>
                            <body>
                                <h1>Hello, World!</h1>
                                <p>This is a simple example of an HTML document.</p>
                            <body>
                        </html>

In this example:

  • The <!DOCTYPE html> ensures the browser knows it's working with an HTML5 document.
  • The <html> tag wraps all other HTML content.
  • The <head> section includes metadata, and the <title> element defines the page's title that appears in the browser tab.
  • The <body> contains the actual content that will be visible on the webpage, like the heading (<h1>) and paragraph (<p>).

HTML Elements and Tags

Difference Between Opening and Closing Tags

HTML elements are represented by tags. Most tags come in pairs: an opening tag and a closing tag. The opening tag starts the element, while the closing tag, which includes a forward slash /, ends it. For example, a paragraph is enclosed between <p> and </p>.

Block-level vs. Inline Elements

HTML elements can be classified either block-level or inline elements.

  • Block-level elements(e.g,<div>,<p>,<h1>) occupy the full width of their container, creating a new line before and after the element.
  • Inline elements(e.g,<span>,<a>,<strong>) do not start on a new line and only take up as much width as necessary, allowing other elements to appear next to them.

Self-closing Tags

Some HTML elements do not require closing tags because they don't enclose any content. These are called self-closing tags. Examples include <img/> for embedding images and <br /> for line breaks. Self-closing tags end with a forward slash (/) before the closing angle bracket.

Example:

For a quick illustration, here's a simple example:

  • <p>This is a paragraph.</p>: This is an example of an opening and closing tag.
  • <img src="image.jpg" alt="Example image"/>: This is an example of self-closing tag.

HTML Attributes

Definition of Attributes

HTML attributes are additional information provided inside an HTML tag that modify the behavior or appearance of an element. Attributes typically come in key-value pairs, where the key is the attribute name and the value provides specific details. For example, in <a href="https://example.com">, href is the attribute name and "https://example.com" is the value.

Common Attributes

There are many attributes available in HTML, but some of the most common ones include:

  • href: Used in <a> tags to define the link destination.
  • src: Used in <img> tags to specify the source file of an image.
  • alt: Provides alternative text for images, improving accessibility.
  • id: Assigns a unique identifier to an element.
  • class: Groups elements together for styling purposes.
  • style: Inline CSS for styling individual elements.

Here's an example of attributes in action:
  • <a href="https://example.com" target="_blank">Visit Example</a>:: The href attribute specifies the link URL, and the target attribute ensures the link opens in a new tab.
  • <img src="image.jpg" alt="A description of the image">: The src attribute defines the image file, and the alt attribute provides descriptive text.

HTML Forms

Structure of a Form

An HTML form is a section of a webpage used to collect user input and send it to a server. The basic structure of a form includes the <form> tag, which wraps all the input elements, and the action attribute that defines where the form data should be sent. Inside the form, you can include various input elements like text fields, buttons, checkboxes, and more.

Different Input Types

HTML offers a wide range of input types to handle different kinds of data. Some common input types include:

  • text: A simple text input field.
  • email: A field specifically for email addresses (validates the input format).
  • password: A field that hides the input for password entry.
  • checkbox: Allows users to select one or more options.
  • radio: Allows users to select one option from a group.
  • submit: A button that sends the form data to the server.

Example

Here's an example of a simple form with different input types: 

                        
                            <form action="/submit" method="POST">
                                <label for="name">Name:</label>
                                <input type="text" id="name" name="name">
                                
                                <label for="email">Email:</label>
                                <input type="email" id="email" name="email">
                                
                                <label for="password">Password:</label>
                                <input type="password" id="password" name="password">
                                
                                <label for="subscribe">Subscribe to newsletter:</label>
                                <input type="checkbox" id="subscribe" name="subscribe">
                                
                                <input type="submit" value="Submit">
                            </form>
In this example:
  • The form collects a name, email, password, and subscription preference.
  • It uses a combination of text, email, password, checkbox, and submit input types.

HTML Semantic Elements

What Are Semantic Elements?

Semantic elements in HTML are tags that clearly describe their meaning and purpose in a web document. Unlike non-semantic elements like <div> or <span>, semantic elements convey the role of the content inside them, making the structure of the webpage more meaningful and easier to understand.

Why Semantic Elements Improve Readability

Semantic elements improve the readability of code for both humans and search engines:

  • For Humans: Developers can easily understand the structure of a webpage when tags like <header>, <article>, and <footer> are used. This makes collaboration and maintenance more efficient.
  • For Search Engines: Search engines use semantic elements to better index and rank websites. By using tags like <article>, <section>, and <nav>, search engines can more accurately interpret the importance of content, which can improve SEO.

Examples of Semantic Elements

  • <header>: Represents the header section of a page or a section of a document.
  • <nav>: Defines a set of navigation links.
  • <article>: Specifies independent, self-contained content such as a blog post or news article.
  • <section>: Groups related content together, often used for different sections of a page.
  • <footer>: Represents the footer of a page or a section, typically containing copyright info or contact details.

Example:

                        
                            <article>
                                <header>
                                    <h1>Understanding Semantic Elements</h1>
                                    <p>Published on September 30, 2024</p>
                                </header>
                                <section>
                                    <p>Semantic elements help improve both code readability and SEO...</p>
                                </section>
                                <footer>
                                    <p>Author: John Doe</p>
                                </footer>
                            </article>
In this example:
  • The <article> tag wraps the content, identifying it as an independent piece of information.
  • The <header>, <section>, and <footer> tags further organize the content, making it easier to read and index.
